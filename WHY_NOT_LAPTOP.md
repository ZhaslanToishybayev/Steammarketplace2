# 🤔 Почему нельзя напрямую через ноутбук?

## 🚫 ПРОБЛЕМЫ С ПРЯМЫМ ДОСТУПОМ

### 1. **Серый IP (NAT)**
```
Ваш ноутбук → Роутер (192.168.1.x) → Провайдер → Интернет
                ↑
            Серый IP (не публичный)

Домен sgomarket.com → DNS → 45.140.25.16 → Роутер (не пускает)
```

**Что происходит:**
- DNS указывает на `45.140.25.16` (ваш публичный IP)
- Но этот IP принадлежит провайдеру, а не вам напрямую
- Роутер не знает, что делать с входящими запросами
- Блокирует все порты (80, 443) по умолчанию

### 2. **Роутер блокирует порты**

**Почему:**
- Домашние роутеры защищают от внешних атак
- Порты 80 (HTTP) и 443 (HTTPS) заблокированы по умолчанию
- Нужен **Port Forwarding** (проброс портов)

**Настройка Port Forwarding:**
```
Роутер → NAT → Port Forwarding:
External Port 80 → 192.168.1.XXX:80
External Port 443 → 192.168.1.XXX:443
```

**Проблемы:**
- Нужны админские права роутера
- Сложная настройка для новичков
- Не всегда работает (провайдер может блокировать)
- Нестабильное соединение

### 3. **Провайдер блокирует веб-порты**

**Почему:**
- Провайдеры блокируют порты 80/443 для домашних пользователей
- Снижает нагрузку, защищает от хостинга
- Невозможно обойти без VPN/VPS

**Проверка:**
```bash
# Если не работает, значит провайдер блокирует
curl http://45.140.25.16:80
curl https://45.140.25.16:443
```
